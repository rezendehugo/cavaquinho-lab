# Política de segurança

## Versões suportadas

Enquanto o projeto estiver na série `0.x`, somente a versão publicada mais recente recebe correções de segurança.

## Relatando uma vulnerabilidade

Não abra uma issue pública com detalhes exploráveis. Use o recurso privado de advisories do GitHub:

https://github.com/rezendehugo/cavaquinho-lab/security/advisories/new

Inclua impacto, passos mínimos para reprodução e uma forma segura de validar a correção. Não envie credenciais reais ou dados pessoais.

O projeto procurará confirmar o recebimento e coordenar divulgação e correção antes de tornar os detalhes públicos. Como projeto mantido de forma voluntária, não há SLA garantido.
